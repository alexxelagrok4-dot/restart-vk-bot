# VK-бот re:старт — деплой на постоянный сервер

Это облачная версия VK-бота для тестового сообщества. Она читает секреты из переменных окружения и подходит для запуска на Render Background Worker, Railway, Fly.io или VPS.

## Рекомендуемый тестовый вариант

Render Background Worker.

Почему:

- простой деплой из GitHub;
- поддерживает Docker;
- есть free/hobby-уровень для тестов;
- логи видны в веб-кабинете;
- не нужно держать компьютер включённым.

## Что нужно заранее

1. Новый VK access token сообщества с правами:
   - messages;
   - manage;
   - wall, если потом будем публиковать;
   - photos, если потом будем грузить фото.

2. ID сообщества:
   - для теста: `240289560`.

3. ID администратора для заявок:
   - можно использовать твой VK ID: `1103900131`;
   - администратор должен сначала написать в сообщения сообщества, чтобы сообщество могло отправлять ему уведомления.

## Переменные окружения

На сервере нужно задать:

```env
VK_ACCESS_TOKEN=новый_токен_сообщества
VK_GROUP_ID=240289560
ADMIN_PEER_ID=1103900131
```

Токен не хранить в коде и не отправлять в чат.

## Деплой на Render

1. Создать аккаунт на https://render.com
2. Залить папку проекта в GitHub.
3. В Render нажать `New` → `Blueprint` или `Background Worker`.
4. Выбрать репозиторий.
5. Указать путь к Dockerfile:

```text
Таск/re_start_platform_repack/vk_bot_deploy/Dockerfile
```

Если Render не принимает кириллический путь, лучше скопировать папку `vk_bot_deploy` в корень отдельного репозитория.

6. Добавить Environment Variables:
   - `VK_ACCESS_TOKEN`
   - `VK_GROUP_ID`
   - `ADMIN_PEER_ID`

7. Нажать Deploy.

## Как проверить

1. Открыть тестовое сообщество:

https://vk.com/club240289560

2. Написать в сообщения:

```text
Привет
```

3. Потом:

```text
Записаться
```

4. Пройти 5 вопросов.

5. Проверить, что администратору пришло уведомление.

## Важное ограничение

Файл `leads.jsonl` внутри бесплатного контейнера может не быть постоянным хранилищем. Поэтому для боевой версии заявки лучше отправлять:

- в Telegram-чат администратора;
- в Google Таблицу;
- в CRM;
- или хотя бы в VK-сообщение администратору через `ADMIN_PEER_ID`.

В этой версии уже есть отправка заявки администратору, если задан `ADMIN_PEER_ID`.
